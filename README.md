# Summary
Building on last week's focus on debugging and critically evaluating AI-generated code, this Week 3 ByteBites Tinker activity pushed students a step further — designing and building the core backend architecture of a campus food ordering app using Python classes and simple algorithms, with AI serving as a collaborative design partner throughout. The key concept students needed to understand was how to prompt AI precisely and intentionally, especially when translating vague feature requests into structured components like UML diagrams and class scaffolds, rather than accepting whatever was generated at face value. Students are most likely to struggle with prompting AI effectively for more complex tasks like class and template generation, as well as making sure to evaluate the AI-generated work before publishing. AI was genuinely helpful for generating baseline templates and scaffolds to build off of, but students should stay critically engaged throughout. As a TF, I would guide students by asking them to articulate what they expect each class to do before touching any AI tool — reinforcing the same "reason first, then use AI" mindset we emphasized last week.
